# 面向恒温控制场景的边缘控制—数据枢纽—应用决策三层架构系统设计与实现

本仓库用于承载本科毕业设计的完整工程实现、设计文档、仿真模型、实验记录与论文支撑材料。

项目核心目标不是单独实现某个 AI 模型，而是围绕恒温控制场景，完成一个具备工程可落地性的三层架构系统：

1. 边缘控制层：负责温度采集、PWM 控制、设备状态输出与本地闭环。
2. 数据枢纽层：负责数据接入、存储、消息流转与系统解耦。
3. 应用决策层：负责可视化、参数配置、运行监控与后续优化扩展。

当前阶段工作重点放在边缘控制层与仿真系统，优先完成“可演示、可验证、可扩展”的闭环温控节点。

## 1. 项目背景

传统温控系统往往只关注局部控制逻辑，缺少从控制节点、数据传输到上层应用的一体化工程设计。为了更贴近真实工业或智能设备场景，本项目采用三层架构进行设计，使控制、数据与应用解耦，便于后续扩展、实验记录与论文撰写。

本课题选用 ESP32 作为边缘主控，以 DS18B20 作为温度采集传感器，以 PWM 作为执行控制输出形式，先在 Wokwi 平台完成闭环仿真，再逐步抽象为真实硬件实现方案。

## 2. 当前已确定方案

### 2.1 硬件与控制基础

- 主控芯片：ESP32
- 主测温传感器：DS18B20
- 执行输出：PWM
- 功率驱动思路：MOSFET 低边驱动
- 控制周期：约 1 s
- 目标温区：25 C ~ 45 C

### 2.2 当前已完成内容

- ESP32 最小运行验证
- 串口输出
- DS18B20 温度读取
- LED 状态指示
- PWM 输出测试
- 基于目标温度的简单比例控制 V1

### 2.3 当前控制目标

- 温度测量精度目标：±0.5 C 量级
- 稳态控制误差目标：<= ±1.0 C
- 输出控制形式：PWM
- 当前阶段算法优先级：阈值控制 -> 比例控制 -> 简化 PID

## 4. 各目录职责

### `docs/`

保存毕业设计说明性文档，包括：

- 系统总体架构
- 模块划分说明
- 接口设计
- 开发计划
- 实验记录模板

### `hardware/`

保存真实硬件相关内容，包括：

- ESP32 引脚分配
- DS18B20 接线方案
- PWM 驱动与 MOSFET 低边驱动说明
- 后续原理图、接线图与器件选型说明

### `simulator/`

保存仿真相关内容，包括：

- Wokwi 工程
- 虚拟热惯性模型
- 闭环温控演示方案
- 仿真配置与说明文档

### `firmware/`

保存 ESP32 固件代码，包括：

- 传感器采集
- 控制器实现
- PWM 输出
- 状态打印与调试接口
- 后续通信接口封装

### `edge-node/`

保存边缘控制节点的模块化抽象，用于把“单片机代码”逐步整理成“节点工程”：

- 节点职责定义
- 数据结构定义
- 控制流程说明
- 与数据枢纽层的接口预留

### `experiments/`

保存实验过程资料，包括：

- 控制参数记录
- 阶跃响应实验
- 稳态误差实验
- 仿真结果与截图说明

### `scripts/`

保存辅助工具，例如：

- 实验数据整理脚本
- 串口日志解析脚本
- CSV 转换与绘图脚本
