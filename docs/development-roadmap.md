# 开发路线图

## 阶段 1：仓库与工程骨架初始化

目标：

- 建立统一目录结构
- 明确模块边界
- 建立基础文档

输出物：

- 初版 README
- 各目录职责说明
- 架构总览文档

## 阶段 2：边缘控制节点 V1 整理

目标：

- 整理现有 ESP32 与 DS18B20 代码
- 统一参数命名与日志输出
- 固化最小闭环运行版本

输出物：

- 固件目录初版
- 引脚与参数说明
- 日志字段说明

## 阶段 3：边缘控制节点 V2 与虚拟热模型

目标：

- 在 Wokwi 中建立热惯性模型
- 形成 PWM 输出影响温度反馈的闭环
- 实现更适合演示与实验记录的控制节点版本

输出物：

- 仿真工程整理版
- 热模型设计说明
- V2 控制流程说明

## 阶段 4：实验记录与性能分析

目标：

- 记录不同目标温度与控制参数下的响应结果
- 分析稳态误差与调节过程
- 为论文图表提供原始数据

输出物：

- 实验日志
- 数据表格
- 曲线图与现象总结

## 阶段 5：数据枢纽层与上层应用扩展

目标：

- 增加数据上传与转发能力
- 引入存储或消息通道
- 完成可视化与参数配置界面

输出物：

- 数据链路说明
- 上层应用原型
- 三层联调说明
