# Stage Experiment Record V1

## 1. Basic Information

- Project: Design and Implementation of a Three-Layer Architecture for Edge Control, Data Hub, and Application Decision in Constant Temperature Scenarios
- Experiment platform: Wokwi
- Current focus: edge-layer temperature control node simulation
- Controller hardware platform: ESP32
- Physical sensor in simulation: DS18B20
- Control output: PWM on GPIO18
- Status indicator: heartbeat LED on GPIO2
- Controlled process in the current closed loop: first-order virtual thermal model
- Target temperature: 35.0 C
- Ambient temperature in the thermal model: 22.0 C
- Control period: 1 s

This document records the current staged experimental results of the temperature-control simulation module. It is intended to serve as an engineering record for later thesis writing, comparison, and extension.

## 2. System Description

At the current stage, the simulation system contains the following parts:

- ESP32 control node
- DS18B20 temperature acquisition
- PWM output on GPIO18
- GPIO2 heartbeat LED
- first-order thermal inertia model
- serial runtime log output
- CSV-style log output for later analysis

The current system uses two temperature-related quantities:

- `sensor_temp_c`: the physical DS18B20 reading in the Wokwi circuit, used as a reference value
- `sim_temp_c`: the virtual controlled temperature generated by the thermal model and used by the controller for closed-loop regulation

This distinction is important for experiment interpretation. At the current stage, the control loop acts on the simulated thermal state rather than on a sensor value that physically changes with heating power.

## 3. Staged Experiment Records

### Experiment 1: P-Control Closed-Loop Verification (V2)

#### Objective

To verify that the system can move from interface validation to a basic closed-loop temperature-control process by introducing a first-order virtual thermal model and a proportional controller.

#### Control Method

- controller type: proportional control
- thermal model: first-order heating and cooling model
- actuator output: PWM

#### Key Observations

- The system could form a clear closed loop.
- The simulated temperature rose from the initial value toward the target.
- The PWM duty cycle was large in the early stage and decreased as the temperature approached the target.
- The system response was direct and easy to observe.
- A visible steady-state error remained near the target temperature.

#### Result Analysis

Under proportional-only control, the control output depends directly on the instantaneous error. As the simulated temperature approaches the target, the error becomes smaller, which causes the PWM output to decrease. Because a nonzero control effort is still required to balance cooling loss, the system stabilizes slightly below the target temperature.

This is a typical steady-state error phenomenon in proportional control and matches the expected control behavior.

#### Stage Conclusion

V2 successfully completed the transition from signal-path verification to closed-loop process verification. It demonstrated that the simulation framework, thermal model, PWM output path, and runtime logging could work together, but it also showed that proportional control alone was not sufficient to eliminate steady-state error.

### Experiment 2: Initial PI Control (V3)

#### Objective

To reduce the steady-state error observed in V2 by introducing an integral term into the controller while preserving a simple and explainable control structure.

#### Control Method

- controller type: PI control
- integral state: accumulated error with bounded range
- output limit: PWM duty cycle clamped to 0 to 255

#### Key Observations

- The steady-state error was reduced significantly compared with V2.
- The simulated temperature could move beyond 35.0 C.
- A small overshoot appeared during the regulation process.
- The integral term accumulated quickly in the early stage and recovered more slowly near the target.

#### Result Analysis

The integral term provides additional control effort based on the history of the error, which helps compensate for the steady-state offset left by proportional control. As a result, the controlled temperature can approach the target much more closely.

However, because the integral term grew relatively fast during the high-error stage, the control output remained relatively strong even when the system was already close to the target. This led to mild overshoot and slower integral recovery.

#### Stage Conclusion

V3 confirmed that PI control is effective for reducing steady-state error in the current virtual thermal system. It also exposed the next tuning problem: the controller had better accuracy than V2, but its transient response was not yet the most balanced form for demonstration and experiment reporting.

### Experiment 3: Tuned PI Control (V3.1)

#### Objective

To improve the transient behavior of the initial PI controller, reduce overshoot, and make the convergence process smoother while preserving the same simulation architecture.

#### Control Method

- controller type: tuned PI control
- tuning direction:
  - keep proportional action relatively stable
  - adjust integral behavior
  - add a simple anti-windup mechanism
- anti-windup strategy:
  - when the output is already saturated and the current error would push it further into saturation, pause integral accumulation for that cycle

#### Key Observations

- The overshoot was reduced compared with the initial PI version.
- The convergence process became smoother.
- The final simulated temperature stabilized very close to the target.
- The typical final regulated temperature was approximately 34.99 C to 35.00 C.
- The controller remained simple enough for undergraduate-level explanation and experiment discussion.

#### Result Analysis

The tuned PI version still uses the same basic control structure, but the integral action is handled more carefully. By preventing continued accumulation when the controller is already saturated in the same direction, the system avoids part of the excessive stored integral action that would otherwise increase overshoot.

This does not turn the controller into a full industrial PID design, but it improves the practical balance between accuracy and transient smoothness in the current simulation stage.

#### Stage Conclusion

V3.1 is the most suitable demonstration version among the three current stages. It preserves the advantages of PI control in reducing steady-state error while providing a more stable and more interpretable transient response.

## 5. Stage Conclusion

At the current stage, the project has completed a meaningful engineering progression in the edge-layer simulation module:

1. a basic proportional closed loop was established
2. the steady-state error issue was identified through experiment
3. a PI controller was introduced to improve accuracy
4. a tuned PI version was obtained to improve overall response quality

These results show that the project has moved beyond simple peripheral validation and now supports structured closed-loop behavior analysis. This provides a solid basis for later experiment design, controller comparison, and thesis discussion.

It should still be emphasized that the current controller is a staged engineering result rather than a final industrial-grade solution. The current value of the work lies in the clarity of the architecture, explainability of the model, reproducibility of the experiments, and extensibility of the implementation.
